     1→# Koru MVP Stabilization Implementation Plan
     2→
     3→> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
     4→
     5→**Goal:** Leave Koru MVP in a buildable, secure, maintainable state with one clear agent flow, the foundation for a schema-driven Presentation Engine, and semantic memory working without local Ollama.
     6→
     7→**Architecture:** Stabilize first (build + security), then unify the two overlapping agent flows around the backend agent (`koruBackend.ts`), refactor the monolithic backend into focused modules, and add a deterministic Presentation Engine that picks visualizers from tool-result schemas rather than from LLM output.
     8→
     9→**Tech Stack:** TypeScript, Vite, React 19, Tailwind 4, Vitest, Playwright.

   963→- [ ] **Step 1: Implement engine**
   964→
   965→```ts
   966→// src/domain/presentation/engine.ts
   967→import { matchSchema } from "../schemas/matcher";
   968→import { VISUALIZER_CATALOG } from "./catalog";
   969→import type { UiBlock } from "../types";
   970→
   971→export interface PresentationInput {
   972→  hint?: { kind: string; confidence: number; enrichment?: Record<string, unknown> };
   973→  toolResults: Array<{ data?: Record<string, unknown>; summary?: string; sources?: unknown[] }>;
   974→}
   975→
   976→export interface PresentationDecision {
   977→  visualizer: string;
   978→  block: UiBlock;
   979→  reason: "schema_match" | "hint_validated" | "fallback_text";
   980→  degraded: boolean;
   981→}
   982→
   983→export function decidePresentation(input: PresentationInput): PresentationDecision | null {
   984→  for (const result of input.toolResults) {
   985→    const schemaMatch = matchSchema(result.data);
   986→    if (!schemaMatch) continue;
   987→
   988→    const visualizer = VISUALIZER_CATALOG[schemaMatch.id];
   989→    if (!visualizer) continue;
   990→
   991→    const hintMatches = input.hint?.kind === schemaMatch.id;
   992→    const degraded = !hintMatches;
   993→    const block = visualizer.render(schemaMatch.data, hintMatches ? input.hint?.enrichment : undefined);
   994→
   995→    return {
   996→      visualizer: schemaMatch.id,
   997→      block,
   998→      reason: hintMatches ? "hint_validated" : "schema_match",
   999→      degraded,
  1000→    };
  1001→  }
  1002→
  1003→  return null;
  1004→}
  1005→```
