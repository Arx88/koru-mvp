🐛 Problemas Encontrados

PROBLEMA #1: Koru aparece 3 VECES en la misma pantalla
Durante la búsqueda (screenshots 1, 2, 3) se ven simultáneamente:

① Mascota del fondo (background del chat) — Koru sentado con un libro/estante
② Avatar circular que yo agregué arriba del panel (.koru-working-avatar) — círculo de 48px
③ Ilustración grande dentro del WorkingPanel (koru-working-illustration) — Koru con estrella/libro

En la referencia: La mascota aparece UNA sola vez (laptop + taza de café).
// TalkOverlay.tsx línea 300-304 — YO agregué el avatar innecesario: <section className="koru-working-panel"> <div className="koru-working-avatar"> ← ❌ ELIMINAR <img src={KORU_AVATAR} /> </div> <img className="koru-working-illustration" /> ← esta SÍ va
PROBLEMA #2: Los 4 chips de estado se ven TODOS iguales
En los screenshots 1, 2 y 3 los 4 chips (Entendiendo, Buscando, Filtrando, Preparando) aparecen renderizados al mismo tiempo sin distinguir claramente cuál está activo vs cuál está done/pending.

En la referencia: El chip activo tiene fondo púrpura, el done tiene check verde, los pending están en gris.
PROBLEMA #3: Barra de progreso NO monótona
A los 5s la barra marca ~80%, pero a los 10s baja a ~40%. El progreso debería subir siempre, nunca bajar.
PROBLEMA #4: Motto cambia aleatoriamente cada pocos segundos
2s: "El camino se hace caminando."
5s: "Acá estamos, dale que sale."
10s: "Tu paciencia se nota. Gracias."

El motto rota con Math.random() en cada render de React, lo que hace que cambie cada vez que el componente se re-renderiza.

En la referencia: El motto es fijo: "Lo que vale no se apura."
// TalkOverlay.tsx línea 297 — rota en CADA render: const motto = MAGIC_MOTTOS[Math.floor(Math.random() * MAGIC_MOTTOS.length)]; // ❌ Debería usar useState o useRef para fijarlo una vez por sesión
PROBLEMA #5: La ilustración es DISTINTA a la referencia
Referencia: Koru con laptop plateada + taza de café amarilla + ojos cerrados.
Actual: Koru con libro morado + estante/cabinet + ojos abiertos.

La función getTaskIllustration() retorna /stitch/working-illustration.png (default) porque el kicker está vacío durante searching.
PROBLEMA #6: El input bar desaparece durante la búsqueda
En los screenshots 1, 2, 3 no se ve el input bar (composer). El WorkingPanel reemplaza el footer completo.

En la referencia: El input bar SÍ está visible abajo del panel.
// TalkOverlay.tsx línea 1263 — WorkingPanel REPLAZA el footer: {processing && ... ? ( <WorkingPanel /> ← reemplaza ) : ( <footer>...</footer> ← oculto durante searching )}
PROBLEMA #7: Cambio brusco de tema al terminar
Durante la búsqueda: fondo ilustrado (claro, con islas flotantes).
Después de responder: fondo navy oscuro sin ilustración.

No hay transición — el cambio es instantáneo y abrupto.
