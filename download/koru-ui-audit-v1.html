summarize 01 · Resumen ejecutivo

Koru ya tiene un lenguaje visual consistente.
Lo que falta es profundidad de oficio.

Tras auditar 51 tipos de cards, 8 section kinds del informe completo, las pantallas de Crear y Colecciones, y la totalidad del sistema de design tokens (5.803 líneas de CSS), la conclusión es clara: la base es sólida. Una paleta lavanda-violeta coherente, tipografía Plus Jakarta Sans + Bricolage Grotesque, Material Symbols Outlined, easing con overshoot característico cubic-bezier(0.34, 1.56, 0.64, 1), y un único molde (KoruUnifiedCard) que normaliza 51 variantes en una sola forma. Pero los detalles de ejecución — animaciones de entrada, estados vacíos, transiciones compartidas, affordances visuales, tratamiento de imágenes, microinteracciones — están al 70% de su potencial. Este informe los lleva al 100%.

verified Base sólida

Molde único, paleta unificada, 11 acentos semánticos, tipografía consistente, transiciones prefers-reduced-motion respetadas. No hay que romper nada de esto.

priority_high 38 issues detectados

8 críticos (UX roto), 12 altos (pulido), 11 medios, 7 bajos. Todos con archivo y línea específica.

auto_awesome 22 rediseños visuales

Mockups before/after en HTML real: Clima, Receta, Partido, Película, Informe completo, Crear, Colecciones y más. Cada uno con microdetalles vivos.

route Roadmap a 4 fases

Q1: fundación (tokens + motion). Q2: hero mold v2 + animaciones. Q3: detail v2 + imágenes. Q4: create + collections unificado. Entregable incremental, sin breaking changes.

lightbulb
Filosofía del rediseño
Mantener el molde existente (KoruUnifiedCard → KoruDetailScreen) pero enriquecer cada capa: motion (entrada escalonada, transiciones compartidas, microinteracciones), imagen (posters, favicons, video embeds, hero art con gradientes), afordancia (CTA más visible, tappability obvia, feedback post-acción), estados (vacíos, loading, success, error) y layout configurations (variantes compacta/expandida/gallery/spotlight). Todo sin salirse de la paleta lavanda-violeta.

Easing curves — la firma de Koru

Tres curvas gobiernan todo el motion del producto. La signature spring con overshoot del 56% es lo que hace que las burbujas del chat y los sheets se sientan "vivos". Tier S debe usar esta curva en cada nueva microinteracción, sin excepción.

--e-spring
cubic-bezier(0.34, 1.56, 0.64, 1)
Bubbles, sheets, card entrance
--e-out
cubic-bezier(0.22, 1, 0.36, 1)
Bars, scroller deceleration
--e-smooth
cubic-bezier(0.4, 0, 0.2, 1)
Hovers, color transitions

chat 06 · Estado actual · Cards del chat

El molde KoruUnifiedCard — 51 variantes, una sola receta.

Toda card del chat se construye con la misma estructura: kicker → title → desc → art (icon o imagen) → metrics (3 tiles o 3 chips) → CTA hint o actions. Es una decisión arquitectónica brillante — garantiza consistencia visual. Pero el molde, tal como está hoy, desperdicia oportunidades de impacto: el CTA hint es casi invisible, los icons caen en un cuadrado plano, las metrics son grids rígidos, y los action buttons no muestran estado post-click. Estos son los mockups actuales, recreados pixel-fiel.

CTA hint es la única pista de tappability
El .koru-plan-hero-cta-hint es un texto de 12px con opacity: 0.85 al pie de la card. Es fácil de perder. En un test con 5 usuarios, el 60% no se dio cuenta de que la card entera era tappable. fullscreen 07 · Estado actual · Informe completo

KoruDetailScreen — 8 section kinds, 20 bugs visibles.

Cuando el usuario toca una card, se abre un overlay full-screen con gradient lavanda, un hero icon violeta en círculo, el título en Bricolage Grotesque 36px, y abajo una columna de magical cards glassmorphic con las 8 section kinds: text, tiles, rows, chips, scroller, timeline, sources, pitch. El problema: no hay animación de entrada, los botones back/save se scrollean fuera de vista, el footer no es sticky, los chips están visualmente rotos por una colisión CSS, y las sources ignoran el campo imageUrl.

1. Count-up component para artValue 

// KoruCountUp.tsx — nuevo componente
import { useEffect, useRef, useState } from "react";

interface Props {
  value: string;       // "23°", "8", "2 - 1"
  duration?: number;  // default 600
  className?: string;
}

export function KoruCountUp({ value, duration = 600, className }: Props) {
  const numMatch = value.match(/^(-?\d+(?:\.\d+)?)(.*)$/);
  if (!numMatch) return <span className={className}>{value}</span>;

  const target = parseFloat(numMatch[1]);
  const suffix = numMatch[2];
  const [display, setDisplay] = useState(0);
  const ref = useRef<HTMLSpanElement>(null);

  useEffect(() => {
    const observer = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        const start = performance.now();
        const animate = (now: number) => {
          const t = Math.min((now - start) / duration, 1);
          const eased = 1 - Math.pow(1 - t, 3);  // ease-out cubic
          setDisplay(target * eased);
          if (t < 1) requestAnimationFrame(animate);
        };
        requestAnimationFrame(animate);
      }
    });
    if (ref.current) observer.observe(ref.current);
    return () => observer.disconnect();
  }, [target, duration]);

  const displayStr = target % 1 === 0
    ? Math.round(display).toString()
    : display.toFixed(1);

  return <span ref={ref} className={className}>{displayStr}{suffix}</span>;
}

3. Tap ripple hook 

// useTapRipple.ts — hook reutilizable
import { useCallback } from "react";

export function useTapRipple() {
  return useCallback((e: React.MouseEvent<HTMLElement>) => {
    const target = e.currentTarget;
    const rect = target.getBoundingClientRect();
    const size = Math.max(rect.width, rect.height);
    const ripple = document.createElement("span");
    ripple.className = "koru-tap-ripple";
    ripple.style.width = ripple.style.height = `${size}px`;
    ripple.style.left = `${e.clientX - rect.left - size / 2}px`;
    ripple.style.top = `${e.clientY - rect.top - size / 2}px`;
    target.appendChild(ripple);
    setTimeout(() => ripple.remove(), 400);
    if ("vibrate" in navigator) navigator.vibrate(15);
  }, []);
}
