🔑 Reglas de prioridad (cuando hay overlap)

  • 1. Procesamiento > Topic > Idle. Si Koru está guardando memoria pero hablando de recetas, gana "memoria" (acción > tema).
  • 2. Idle solo pisa si no hay nada activo. El timer de 2 min arranca solo cuando el chat está estable.
  • 3. Topic persiste 30s post-respuesta. Después de contestar sobre recetas, el fondo se queda en "recetas" 30s antes de volver a "escuchando".
  • 4. Celebrando es prioritario. Cuando el proactive engine pega (partido, lluvia, etc.), pisa todo por 8s.

// src/states/registry.ts // Mapa canonical: estado → asset. Single source of truth. export type KoruState = | 'escuchando' | 'trabajando' | 'buscando' | 'memoria' | 'construyendo' | 'habitos' | 'productos' | 'recetas' | 'durmiendo' | 'celebrando'; interface StateAsset { type: 'video' | 'image'; src: string; loop?: boolean; // fallback si el video no carga (país con data, browser viejo) poster?: string; } export const STATE_REGISTRY: Record<KoruState, StateAsset> = { escuchando: { type: 'video', src: '/koru-states/estado-durmiendo.mp4', poster: '/koru-states/estado-productos.png' }, trabajando: { type: 'video', src: '/koru-states/estado-trabajando.mp4' }, buscando: { type: 'video', src: '/koru-states/estado-buscando.mp4' }, memoria: { type: 'video', src: '/koru-states/estado-memoria.mp4' }, construyendo: { type: 'image', src: '/koru-states/estado-construyendo.png' }, habitos: { type: 'image', src: '/koru-states/estado-habitos.png' }, productos: { type: 'image', src: '/koru-states/estado-productos.png' }, recetas: { type: 'image', src: '/koru-states/estado-recetas.png' }, durmiendo: { type: 'video', src: '/koru-states/estado-durmiendo.mp4' }, celebrando: { type: 'image', src: '/koru-states/estado-productos.png' }, // fallback };
// src/states/BackgroundLayer.tsx // Renderiza TODOS los assets en capas. Solo el activo tiene opacity:1. // Crossfade CSS = transición suave sin re-mounts. import { useState, useEffect, useRef } from 'react'; import { STATE_REGISTRY, KoruState } from './registry'; export function BackgroundLayer({ state }: { state: KoruState }) { const [active, setActive] = useState<KoruState>(state); const videoRefs = useRef<Record<string, HTMLVideoElement>>({}); useEffect(() => { setActive(state); // arranca el video del estado nuevo, pausa los demás Object.entries(videoRefs.current).forEach(([s, v]) => { if (s === state) v.play().catch(() => {}); else v.pause(); }); }, [state]); return ( <div className="koru-bg-stack"> {Object.entries(STATE_REGISTRY).map(([s, asset]) => ( <div key={s} className={`koru-bg-layer ${s === active ? 'active' : ''}`} style={{ opacity: s === active ? 1 : 0 }}> {asset.type === 'video' ? ( <video ref={el => el && (videoRefs.current[s] = el)} src={asset.src} muted loop playsInline preload="auto" /> ) : ( <img src={asset.src} alt="" /> )} </div> ))} </div> ); } /* CSS: .koru-bg-layer { position:absolute; inset:0; transition: opacity 800ms ease; } */
// src/states/useKoruState.ts // Hook que escucha el stream SSE del backend y devuelve el estado actual. import { useState, useEffect } from 'react'; import { KoruState } from './registry'; import { applyTransition } from './transitions'; export function useKoruState(sessionId: string) { const [state, setState] = useState<KoruState>('escuchando'); useEffect(() => { const es = new EventSource(`/api/chat/events?session=${sessionId}`); es.addEventListener('state', (e) => { const next = JSON.parse(e.data).state as KoruState; setState(prev => applyTransition(prev, next)); // respeta prioridades }); return () => es.close(); }, [sessionId]); return state; } // Uso en App.tsx: // const koruState = useKoruState(sessionId); // <BackgroundLayer state={koruState} /> // <ChatUI state={koruState} /> // el pill del header lo usa también
// src/api/chat.ts (extracto · emite eventos de estado por SSE) // El backend ya sabe en todo momento qué está pasando. // Solo necesita emitir 'state' events además de los 'message' events. async function handleChat(req, res) { const { message, sessionId } = await req.json(); // 1) Usuario mandó mensaje → Koru despierta emitState(res, 'trabajando'); const llmStream = await callLLM(message); for await (const chunk of llmStream) { if (chunk.tool_calls?.length) { const tool = chunk.tool_calls[0].function.name; if (['weather', 'sports', 'web_search'].includes(tool)) { emitState(res, 'buscando'); // 🔍 } else if (['save_expense', 'save_reminder'].includes(tool)) { emitState(res, 'memoria'); // 📝 } await runTool(chunk.tool_calls[0]); } if (chunk.content?.length > 800) { emitState(res, 'construyendo'); // 🧱 respuesta larga } res.write(`data: ${JSON.stringify({ type: 'message', chunk })}\n\n`); } // 3) Topic detection post-respuesta (LLM tag rápido o regex) const topic = detectTopic(message); if (topic) emitState(res, topic); // 'recetas' | 'habitos' | 'productos' // 4) Vuelve a escuchando después de 30s sin actividad scheduleIdle(sessionId, 30_000, () => emitState(res, 'escuchando')); scheduleSleep(sessionId, 120_000, () => emitState(res, 'durmiendo')); } function emitState(res, state) { res.write(`event: state\ndata: ${JSON.stringify({ state })}\n\n`); } function detectTopic(msg): KoruState | null { const t = msg.toLowerCase(); if (/(receta|cocinar|comida|ingredient|asado|pizza|hambre)/.test(t)) return 'recetas'; if (/(gym|gimna|deporte|correr|peso|rutina|entren|futbol)/.test(t)) return 'habitos'; if (/(compr|gasto|gasté|mercado|precio|producto|plata)/.test(t)) return 'productos'; return null; }
// src/states/transitions.ts // Reglas de prioridad: ¿cuándo se permite el cambio prev → next? // Evita flicker entre estados que pelean. import { KoruState } from './registry'; const PRIORITY: Record<KoruState, number> = { celebrando: 100, // pisa todo (proactive engine) memoria: 80, // guardando > todo lo demás buscando: 70, construyendo: 60, trabajando: 50, recetas: 30, habitos: 30, productos: 30, escuchando: 10, durmiendo: 5, }; const MIN_DURATION_MS: Partial<Record<KoruState, number>> = { // anti-flicker: un estado se queda mínimo X ms antes de poder cambiar celebrando: 8_000, memoria: 3_000, buscando: 2_000, construyendo: 3_000, trabajando: 1_500, }; let lastChange: { state: KoruState; at: number } = { state: 'escuchando', at: Date.now() }; export function applyTransition(prev: KoruState, next: KoruState): KoruState { const now = Date.now(); const elapsed = now - lastChange.at; const minDur = MIN_DURATION_MS[prev] ?? 0; // Regla 1: mismo estado, no hacer nada if (prev === next) return prev; // Regla 2: anti-flicker — si el estado actual no cumplió su mínima, mantener if (elapsed < minDur && PRIORITY[next] <= PRIORITY[prev]) return prev; // Regla 3: si next tiene mayor prioridad, pisa siempre if (PRIORITY[next] > PRIORITY[prev]) { lastChange = { state: next, at: now }; return next; } // Regla 4: misma o menor prioridad, respeta min duration if (elapsed >= minDur) { lastChange = { state: next, at: now }; return next; } return prev; }

Performance y delivery

Para que los videos pesen poco y carguen rápido en mobile con data.

📦 Assets pre-convertidos

Los MP4 originales eran HEVC 2160×3840 (~13 MB c/u, no soportados en Chrome/Firefox). Los convertimos a H.264 · 540×960 · CRF 28. Tamaño final: ~280–600 KB por video. Carga instantánea incluso en 3G.

⚡ preload estratégico

preload="auto" en los 4 videos. Se cargan en paralelo al abrir el chat. Peso total del state pack: ~1.4 MB (menos que una foto de Instagram).

🎞️ Crossfade CSS, no re-mount

Todos los assets viven en capas absolutas. Solo cambia opacity con transition 800ms. El video no se reinicia al volver al estado — sigue donde estaba.

📱 Battery-friendly

Solo el video activo reproduce. Los demás están pausados via v.pause(). En estado durmiendo, framerate perceived es bajo (movimiento lento del personaje).

📶 Fallback a PNG

Si el dispositivo está en data-saver o el video falla, el poster PNG toma el lugar. Mismo estado visual, sin movimiento pero sin romper la UX.

🎨 Asset pipeline futuro

Cuando animes los 4 PNG estáticos (Construyendo, Hábitos, Productos, Recetas), el cambio en código es solo actualizar STATE_REGISTRY: { type: 'image' } → { type: 'video' }. Cero cambios en el renderer.

Roadmap de implementación

3 pasos para tener esto andando en producción en el repo Arx88/koru-mvp.

1
Subir assets al repo

Copiar los 8 archivos a public/koru-states/ en la rama main. Commit directo, deploy automático en Render.

⏱️ 5 min · sin risk

2
Emitir estados desde el backend

En api/chat.ts, agregar emitState() en los 4 puntos clave: typing start, tool call, tool save, response end. + scheduler idle/sleep.

⏱️ 30 min · risk bajo

3
Reemplazar fondo estático

En el frontend (HTML/JS actual), reemplazar el <img src="fondo.jpg"> por el BackgroundLayer con state store mínimo. Crossfade CSS.

⏱️ 1-2 h · risk medio

✅ Listo para producción

  • Frontend actual (HTML/JS vanilla): se puede integrar tal cual con un simple store de estado y setInterval para idle/sleep.
  • Backend actual (Nemotron Ultra): los eventos de tool call ya existen en el loop de streaming — solo hay que emitirlos como eventos SSE aparte.
  • El bug del weather no bloquea esto: los fondos son independientes del tool. Cuando weather funcione, automáticamente va a mostrar "buscando" cuando se llame.
