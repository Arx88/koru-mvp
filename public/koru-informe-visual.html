Identidad y tesis

¿Qué es Koru?

Tres ideas centrales en el código — no marketing, sino implementadas. Cada una con su módulo y su restricción explícita.

spa

Metáfora del jardín

Semilla que crece con confianza confirmada

Koru NO busca retención artificial. Las forbiddenPhrases en soul.ts prohíben explícitamente "te extrane", "no me abandones", "siempre estare aqui para ti" — el comportamiento pegajoso que diseñan otros asistentes para capturar usuarios.

El usuario es el jardinero: rega (confirma), poda (descarta). Koru crece solo con lo que el usuario decide confiarle.

verified_user

Memoria confirmable

Candidate → confirmed → rejected

Cuando Koru aprende algo, lo guarda como candidate. Solo se promueve a confirmed cuando el usuario lo confirma. Las memorias confirmed se usan para personalizar; las candidate se usan con cautela y NUNCA se presentan como ciertas.

Esto es lo que lo distingue de un RAG ingenuo: el usuario controla qué se queda.

fact_check

Anti-alucinación por diseño

Cada dato con cita literal

structureExtractor valida cada dato contra una cita literal del source. Si la tool devuelve status: failed o not_configured, el system prompt dice explícitamente "NO inventes los datos".

Es el módulo mejor diseñado del proyecto según la auditoría — referencia de calidad profesional.

Dónde aparece "Trabajando..."

En el paso 4 cuando el pedido requiere investigación (deliverable). El WorkingPanel muestra fases reales del pipeline: entendí → busqué → comparando → redactando. NO aparece para saludos ni pedidos triviales.

Dónde aparece la carta unificada

En el paso 6 cuando llega un chunk con deliverable status:ready. Se renderiza con KoruUnifiedCard (kicker → título → contenido → CTA). El CTA abre KoruDetailScreen con módulos magical-card.

Persistencia — 100% client-side

IndexedDB koru-local-first (store state, key current) + localStorage legacy koru.mvp.state.v1. El server (Vite middleware) es stateless, solo mantiene providers en memoria. Un solo molde

Detalle es pantalla

Búsqueda es estado
